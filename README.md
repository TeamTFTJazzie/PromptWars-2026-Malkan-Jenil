# Chosen Vertical
Emergency & Medical Incident Triage.
This application acts as a critical triage point designed to ingest frantic, unstructured emergency inputs natively through voice dictation, typed text panic, or raw photo uploads. It rapidly converts these messy, real-world signals into clean, structured dispatch data prioritizing urgency, identifying location, and suggesting life-saving 3-to-5 step action protocols securely.

# Approach and Logic
The architectural priority was performance, strict security, and comprehensive WCAG AAA semantic integrity. 
- **Tech Stack**: Enterprise Vite + React scaffold strictly adhering to `/components`, `/services`, `/hooks`, `/tests`, and `/utils` separation of concerns.
- **Performance**: Deep integration of `React.useMemo` for any derived DOM purifications or data computations, and `React.useCallback` for static event delegates so that UI renders never needlessly block the main thread during high-stakes triage.
- **Security Protocols**: Broad regex sweeps filter naive injection vectors (`<script>`, SQL logic), while all unstructured HTML generated by the AI is mathematically sanitized via `DOMPurify` before entering the DOM. The Gemini key (`VITE_GEMINI_API_KEY`) is securely isolated into environment parameters and excluded strictly from the source code.
- **Accessibility**: High-contrast, custom `#0A1628` styling, rigorous tab indexing, and `aria-live="polite"` attributes guarantee accessibility for users physically or cognitively distressed by the emergency.

# How the Solution Works
1. **Multimodal Input Capture**: Distressed users utilize `TriageForm` to provide context via Web Speech API dictation (custom hook), standard text, or photo uploads (bounding file/size limitations in `utils/validation.js`).
2. **AI Processing**: The `geminiService` strictly directs the `@google/genai` sdk to act as an Emergency Dispatcher. It requires the raw string or parsed base64 image and mandates a hard `application/json` formatting response matching two structural keys: `actionPlan` (HTML representation) and `dispatchData`.
3. **Rendering Lifecycle**: Responses hit the global `App` state. The derived state activates `ActionPlanDisplay` to unwrap the safely sanitized DOM output and maps the inferred location dynamically using a standard <iframe> map protocol to avoid Google authentication restrictions.

# Assumptions Made
1. **Network Constraint**: The solution assumes the user maintains a mobile internet connection capable of transmitting small payloads (<5MB images).
2. **Device Features**: Web Speech API dictate capabilities assume browser-level vendor support (e.g. `webkitSpeechRecognition` availability natively).
3. **Map Fallback**: Given API restrictions on Google Maps natively during the MVP, the embedded <iframe> defaults cleanly to basic URL query encoding without breaking or requiring a key rotation.
